Python Webアプリケーションを公開する方法

概要

Pythonを含むWebアプリケーション（Flask）を公開するには、静的サイトとは異なる手順が必要です。 サーバーサイドでPythonを実行できる環境が必要になります。

重要： 通常のWebホスティング（HTMLファイルのアップロード）では、 Pythonアプリケーションは動作しません。専用のプラットフォームが必要です。

無料で始められる方法

1. Heroku（推奨）

最も人気のPaaSプラットフォーム

  • • 無料プランあり（制限あり）
  • • Gitベースのデプロイ
  • • 自動スケーリング

2. Railway

シンプルで使いやすい

  • • 自動環境検出
  • • 簡単なデプロイ
  • • 無料クレジットあり

3. Render

静的サイトと動的サイト両対応

  • • 無料プランあり
  • • 自動デプロイ
  • • 簡単な設定

有料だが高性能な方法

1. AWS EC2

完全なサーバー制御

  • • 月額$5〜
  • • 高スケーラビリティ
  • • 学習コスト高

2. DigitalOcean

シンプルなVPS

  • • 月額$5〜
  • • ドキュメント充実
  • • 中級者向け

3. Google Cloud

Googleのクラウドプラットフォーム

  • • 無料クレジットあり
  • • 高機能
  • • 複雑な設定

Herokuでのデプロイ手順（推奨）

1. 事前準備

1

Herokuアカウント作成

https://heroku.com でアカウントを作成

2

Heroku CLIインストール

https://devcenter.heroku.com/articles/heroku-cli

3

Gitの設定

Gitがインストールされていることを確認

2. プロジェクトの準備

Procfileを作成

site/Python/timeline/Procfile
web: python app.py

requirements.txtを作成

site/Python/timeline/requirements.txt
Flask==2.3.3

3. デプロイコマンド

コマンドプロンプト / ターミナル
cd site/Python/timeline
git init
git add .
git commit -m "Initial commit"
heroku login
heroku create your-timeline-app
git push heroku main

その他のデプロイ方法

Railway

最も簡単な方法

  1. 1. railway.app でアカウント作成
  2. 2. GitHubと連携
  3. 3. リポジトリを選択
  4. 4. 自動デプロイ完了

Render

静的・動的両対応

  1. 1. render.com でアカウント作成
  2. 2. GitHubと連携
  3. 3. Web Serviceを選択
  4. 4. 設定してデプロイ

重要な注意点

  • • データベース： SQLiteは本番環境では推奨されません。PostgreSQLやMySQLを使用してください
  • • 環境変数： 機密情報は環境変数で管理してください
  • • デバッグモード： 本番環境ではdebug=Falseに設定してください
  • • 静的ファイル： CSSやJSファイルはCDNを使用することを推奨します
